**展示：**

#### 步骤 3-7: CI Pipeline
> "GitHub Actions 自动执行一系列检查和测试："

**逐个讲解：**
- **CI Trigger**: 自动触发构建
- **Install Dependencies**: 安装 Python 依赖

#### 步骤 8-9: Build & Docker

**展示：**
- 指向 "Build App" 和 "Docker Build" 步骤
- 打开 "Docker Container" 卡片
- **重点讲解 Docker 的价值**：

**Docker 的 5 大优势（重点！）：**

1. **Consistency（一致性）**
   > "开发、测试、生产环境完全一致，消除'在我机器上能跑'的问题"

2. **Isolation（隔离性）**
   > "所有依赖都打包在镜像中，不会与系统其他软件冲突"

3. **Portability（可移植性）**
   > "可以在任何安装了 Docker 的平台上运行：Windows、Linux、Mac、云端"

4. **Reproducibility（可重现性）**
   > "每次构建都是完全相同的，确保部署的可靠性"

5. **Efficiency（高效性）**
   > "秒级启动，资源占用少，比虚拟机更轻量"

**展示 Docker 卡片信息：**
- Image 名称和标签
- 从哪个 commit 构建的
- 容器状态（Running）
- 端口映射（8000:8000）
- 创建时间和运行时长

#### 步骤 10: Deploy
> "最后，容器自动部署到生产环境，服务立即可用。"

**展示：**

## ❓ Q&A 准备

### 常见问题及回答

**Q1: 如果测试失败会怎样？**
> A: Pipeline 会停止，不会部署到生产环境。开发者会收到通知，必须修复问题后才能继续。Dashboard 会显示失败的步骤（红色），活动日志会记录失败原因。

**Q2: 部署需要多长时间？**
> A: 从提交代码到部署完成，通常需要 3-5 分钟。这包括所有测试和构建过程。如果只是小改动，可能更快。

**Q3: 可以回滚到之前的版本吗？**
> A: 可以。每个 Docker 镜像都有对应的 commit SHA 标签，可以快速回滚到任何历史版本。

**Q4: 这套系统的成本如何？**
> A: 主要成本是 GitHub Actions 的运行时间（有免费额度）和服务器资源。相比人工部署节省的时间和减少的错误，ROI 非常高。

**Q5: 其他项目可以复用这套流程吗？**
> A: 完全可以。这是一套标准化的 DevOps 流程，只需要调整配置文件就能应用到其他项目。

**Q6: 如何保证安全性？**
> A: 
> - 所有代码都经过审查和测试
> - GitHub 有访问控制和权限管理
> - Docker 镜像签名和扫描
> - 生产环境有防火墙和监控

**Q7: 团队需要学习多久？**
> A: 开发者只需要学会基本的 Git 操作，其他都是自动的。新人通常 1-2 天就能上手。

## 🎯 演示技巧

### 演讲技巧
1. **语速适中**：不要太快，确保观众能跟上
2. **多用比喻**：用生活中的例子解释技术概念
3. **互动提问**：适时询问观众是否有问题
4. **眼神交流**：不要一直盯着屏幕

### 屏幕操作技巧
1. **鼠标指示**：用鼠标圈出重点区域
2. **放大显示**：重要信息可以放大
3. **暂停等待**：给观众时间消化信息
4. **准备备份**：如果网络有问题，准备截图

### 应急预案
1. **Dashboard 无法加载**：使用截图或录屏
2. **实时演示失败**：使用之前准备的成功案例
3. **时间不够**：跳过实时演示，直接展示结果
4. **技术问题**：坦诚说明，强调系统的稳定性
