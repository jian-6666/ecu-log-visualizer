## 🚨 紧急情况处理

### 如果 Docker 无法启动

**方案 A：使用 Python 直接运行**
```bash
python run_server.py
```

**方案 B：展示离线材料**
- 使用准备好的截图
- 播放录制的视频

### 如果端口被占用

```bash
# 查找占用进程
lsof -i :8000  # Mac/Linux
netstat -ano | findstr :8000  # Windows

# 杀死进程或使用其他端口
# 修改 run_server.py 中的端口号
```

### 如果测试失败

**方案 A：跳过测试步骤**
- 说明："测试在之前已经验证通过"
- 继续后续演示

**方案 B：展示之前的测试结果**
- 使用截图或日志文件

### 如果网络问题

**方案 A：使用本地功能**
- 跳过 GitHub/Jenkins 相关演示
- 专注于本地功能

**方案 B：使用离线材料**
- 展示截图说明 CI/CD 流程

### 如果时间不够

**优先展示：**
1. 工程仪表板（最重要）
2. Docker 部署
3. 自动化测试

**可以跳过：**
1. 详细的代码讲解
2. 完整的测试运行
3. 代码改动演示
