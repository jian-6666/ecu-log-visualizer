# ECU 日志文件格式规范

本文档描述 ECU Log Visualizer 支持的日志文件格式。

## 支持的格式

系统支持两种日志文件格式：
- CSV (Comma-Separated Values)
- JSON (JavaScript Object Notation)

## CSV 格式

### 基本结构

CSV 文件必须包含表头行，后跟数据行。

### 必需字段

- `timestamp`: 时间戳，ISO 8601 格式（例如：2024-01-01T00:00:00）

### 可选字段

- 传感器列：任意数量的数值列，列名即为传感器名称
- `status`: 状态字段（例如：OK, WARNING, ERROR）

### 示例

```csv
timestamp,engine_temp,oil_pressure,rpm,throttle_position,battery_voltage,status
2024-01-01T08:00:00,85.2,45.3,800,0.0,12.6,OK
2024-01-01T08:00:01,85.4,45.5,820,2.5,12.6,OK
2024-01-01T08:00:02,85.6,45.7,850,5.0,12.5,OK
2024-01-01T08:00:03,85.9,46.0,900,8.5,12.5,OK
2024-01-01T08:00:04,86.2,46.3,950,12.0,12.5,OK
```

### 格式要求

1. **时间戳格式**：
   - 必须是有效的 ISO 8601 格式
   - 支持的格式示例：
     - `2024-01-01T00:00:00`
     - `2024-01-01T00:00:00Z`
     - `2024-01-01T00:00:00+00:00`

2. **传感器值**：
   - 必须是数值类型（整数或浮点数）
   - 非数值条目将被跳过并记录警告

3. **文件编码**：
   - UTF-8 编码

4. **分隔符**：
   - 逗号 (,)

## JSON 格式

### 基本结构

JSON 文件必须是一个对象数组，每个对象代表一条日志记录。

### 必需字段

- `timestamp`: 时间戳字符串，ISO 8601 格式

### 可选字段

- `sensors`: 对象，包含传感器名称和数值的键值对
- `status`: 状态字符串

### 示例

```json
[
  {
    "timestamp": "2024-01-01T08:00:00",
    "sensors": {
      "engine_temp": 85.2,
      "oil_pressure": 45.3,
      "rpm": 800,
      "throttle_position": 0.0,
      "battery_voltage": 12.6
    },
    "status": "OK"
  },
  {
    "timestamp": "2024-01-01T08:00:01",
    "sensors": {
      "engine_temp": 85.4,
      "oil_pressure": 45.5,
      "rpm": 820,
      "throttle_position": 2.5,
      "battery_voltage": 12.6
    },
    "status": "OK"
  }
]
```

### 格式要求

1. **根元素**：
   - 必须是数组

2. **记录对象**：
   - 每个对象必须包含 `timestamp` 字段
   - `sensors` 对象中的值必须是数值类型

3. **文件编码**：
   - UTF-8 编码

## 数据验证规则

系统在解析日志文件时会执行以下验证：

### 时间戳验证

- 时间戳必须是有效的 ISO 8601 格式
- 如果时间戳乱序，系统会自动排序并记录警告

### 传感器值验证

- 传感器值必须是数值类型
- 有效范围：-1e10 到 1e10
- 非数值条目会被跳过并记录警告

### 文件大小限制

- 最大文件大小：50MB
- 建议最大记录数：1,000,000 条

### 传感器数量限制

- 最大传感器数量：100 个
- 图表同时显示最多 20 个传感器

## 数据质量建议

为获得最佳分析结果，建议：

1. **时间戳连续性**：
   - 保持时间戳按时间顺序排列
   - 避免时间戳重复

2. **数据完整性**：
   - 尽量避免缺失值
   - 确保传感器值在合理范围内

3. **传感器命名**：
   - 使用描述性的传感器名称
   - 避免使用特殊字符
   - 推荐使用下划线分隔单词（例如：engine_temp）

4. **状态字段**：
   - 使用标准化的状态值（例如：OK, WARNING, ERROR）
   - 保持状态值一致性

## 错误处理

### 解析错误

如果文件解析失败，系统会返回包含以下信息的错误：
- 错误类型
- 错误行号（如适用）
- 错误描述

### 警告信息

对于非致命错误（如非数值数据），系统会：
- 跳过有问题的条目
- 在响应中包含警告信息
- 报告跳过的条目数量

## 示例文件

系统提供了示例文件供参考：
- `examples/sample_ecu_log.csv` - CSV 格式示例
- `examples/sample_ecu_log.json` - JSON 格式示例

这些文件包含：
- 100+ 数据点
- 6 种不同传感器
- 正常和异常读数示例
- 不同状态值示例
