## 系统要求

在开始之前，请确保你的电脑上已安装：

- ✅ **Python 3.9 或更高版本**
  - 检查方法：打开命令行，输入 `python --version`
  - 如果没有，请从 https://www.python.org/downloads/ 下载安装

- ✅ **Git**（可选，用于版本控制）
  - 检查方法：打开命令行，输入 `git --version`
  - 如果没有，请从 https://git-scm.com/downloads 下载安装

- ✅ **Docker**（可选，用于容器化部署）
  - 检查方法：打开命令行，输入 `docker --version`
  - 如果没有，请从 https://www.docker.com/get-started 下载安装

## 安装步骤

### 方法一：从 GitHub 克隆（推荐）

#### 步骤 1：打开命令行
- **Windows**：按 `Win + R`，输入 `cmd`，按回车
- **Mac/Linux**：打开 Terminal

#### 步骤 2：克隆项目
```bash
git clone https://github.com/jian-6666/ecu-log-visualizer.git
cd ecu-log-visualizer
```

#### 步骤 3：安装依赖
```bash
pip install -r requirements.txt
```

等待安装完成（可能需要 1-2 分钟）。

## 启动应用

### 方法一：使用 Python 直接运行（最简单）

#### 步骤 1：在项目目录下运行
```bash
python run_server.py
```

#### 步骤 2：看到以下输出表示成功
```
============================================================
ECU Log Visualizer - Backend Server
============================================================
Starting server on http://127.0.0.1:8000
API Documentation: http://127.0.0.1:8000/docs
Alternative Docs: http://127.0.0.1:8000/redoc
Health Check: http://127.0.0.1:8000/health
============================================================
Press CTRL+C to stop the server
============================================================
```

#### 步骤 3：保持命令行窗口打开
**重要**：不要关闭这个窗口，服务器需要一直运行。

---

### 方法二：使用 Docker 运行（可选）

#### 步骤 1：构建 Docker 镜像
```bash
docker build -t ecu-log-visualizer:latest .
```

等待构建完成（第一次可能需要 3-5 分钟）。

#### 步骤 2：运行容器
```bash
docker run -d --name ecu-log-visualizer -p 8000:8000 ecu-log-visualizer:latest
```

#### 步骤 3：检查容器状态
```bash
docker ps
```

应该看到 `ecu-log-visualizer` 容器正在运行。

## 使用主应用

### 步骤 1：打开浏览器

打开你喜欢的浏览器（Chrome、Firefox、Edge 等），访问：
```
http://localhost:8000
```

### 步骤 2：上传 ECU 日志文件

#### 2.1 点击"选择文件"按钮
在页面顶部找到文件上传区域。

#### 2.2 选择示例文件
项目自带了示例文件，位于：
```
examples/sample_ecu_log.csv
```

或者使用你自己的 ECU 日志文件（支持 CSV 和 JSON 格式）。

#### 2.3 点击"上传"按钮
等待几秒钟，文件上传完成。

### 步骤 3：查看数据分析

上传成功后，页面会自动显示：

#### 3.1 统计表格
显示每个传感器的：
- **最小值** - 最低读数
- **最大值** - 最高读数
- **平均值** - 平均读数
- **标准差** - 数据波动程度

#### 3.2 交互式图表
- **缩放**：使用鼠标滚轮放大/缩小
- **平移**：按住鼠标左键拖动
- **悬停**：鼠标移到数据点上查看具体数值

### 步骤 4：使用过滤功能

#### 4.1 选择时间范围
在右侧找到"时间范围"选择器：
- 点击开始时间，选择起始时间
- 点击结束时间，选择结束时间

#### 4.2 选择传感器
在"传感器选择"区域：
- 勾选你想查看的传感器
- 取消勾选你想隐藏的传感器

#### 4.3 应用过滤
点击"应用过滤"按钮，图表和统计会立即更新。

### 步骤 5：导出数据

#### 5.1 点击"导出 CSV"按钮
在页面右侧找到导出按钮。

#### 5.2 保存文件
浏览器会自动下载过滤后的数据为 CSV 文件。

## 常见问题

### Q1: 服务器启动失败，显示"端口被占用"

**问题**：
```
OSError: [Errno 48] Address already in use
```

**解决方法**：

**Windows**：
```bash
# 查找占用 8000 端口的进程
netstat -ano | findstr :8000

# 记下 PID（最后一列的数字），然后杀死进程
taskkill /PID <PID号> /F

# 重新启动服务器
python run_server.py
```

**Mac/Linux**：
```bash
# 查找占用 8000 端口的进程
lsof -i :8000

# 杀死进程（替换 <PID> 为实际的进程 ID）
kill -9 <PID>

# 重新启动服务器
python run_server.py
```

---

### Q2: 上传文件失败

**可能原因**：
1. 文件格式不正确（只支持 CSV 和 JSON）
2. 文件太大（限制 50MB）
3. 文件内容格式不符合要求

**解决方法**：
1. 使用示例文件测试：`examples/sample_ecu_log.csv`
2. 检查文件大小：右键文件 → 属性 → 查看大小
3. 检查文件格式：确保是 CSV 或 JSON 格式

**CSV 文件格式要求**：
```csv
timestamp,sensor_name,value
2024-01-01 10:00:00,temperature,25.5
2024-01-01 10:00:01,pressure,101.3
```

**JSON 文件格式要求**：
```json
[
  {
    "timestamp": "2024-01-01 10:00:00",
    "sensor_name": "temperature",
    "value": 25.5
  }
]
```

---

### Q3: 工程仪表板显示"Loading..."不消失

**可能原因**：
1. 服务器未启动
2. 网络连接问题
3. API 响应超时

**解决方法**：

**步骤 1**：检查服务器是否运行
```bash
curl http://localhost:8000/health
```

应该返回：
```json
{"status":"healthy","service":"ECU Log Visualizer","max_upload_size_mb":50.0}
```

**步骤 2**：检查浏览器控制台
1. 按 F12 打开开发者工具
2. 点击"Console"标签
3. 查看是否有错误信息

**步骤 3**：刷新页面
按 Ctrl+F5（Windows）或 Cmd+Shift+R（Mac）强制刷新。

---

### Q4: GitHub Actions 没有触发

**可能原因**：
1. GitHub Actions 未启用
2. 工作流文件有语法错误
3. 没有推送到正确的分支

**解决方法**：

**步骤 1**：检查 GitHub Actions 是否启用
1. 访问：https://github.com/jian-6666/ecu-log-visualizer
2. 点击"Settings"
3. 点击左侧"Actions" → "General"
4. 确保"Allow all actions and reusable workflows"已选中

**步骤 2**：检查工作流文件
确保 `.github/workflows/ci.yml` 文件存在且格式正确。

**步骤 3**：检查分支
确保推送到 main 或 master 分支：
```bash
git branch  # 查看当前分支
git push origin main  # 推送到 main 分支
```

---

### Q5: Docker 构建失败

**可能原因**：
1. Docker 未启动
2. 磁盘空间不足
3. 网络连接问题

**解决方法**：

**步骤 1**：检查 Docker 是否运行
```bash
docker info
```

如果显示错误，启动 Docker Desktop。

**步骤 2**：检查磁盘空间
```bash
docker system df
```

如果空间不足，清理：
```bash
docker system prune -a
```

**步骤 3**：重新构建
```bash
docker build -t ecu-log-visualizer:latest .
```

---

### Q6: 图表不显示或显示不正确

**可能原因**：
1. 数据格式不正确
2. 浏览器兼容性问题
3. JavaScript 错误

**解决方法**：

**步骤 1**：检查数据
确保上传的文件包含有效的数据。

**步骤 2**：更换浏览器
尝试使用 Chrome 或 Firefox 最新版本。

**步骤 3**：清除缓存
按 Ctrl+Shift+Delete（Windows）或 Cmd+Shift+Delete（Mac），清除浏览器缓存。

```bash
# 启动服务器
python run_server.py

# 停止服务器
# 按 Ctrl+C

# 使用 Docker 启动
docker run -d --name ecu-log-visualizer -p 8000:8000 ecu-log-visualizer:latest

# 使用 Docker 停止
docker stop ecu-log-visualizer
docker rm ecu-log-visualizer
```

### 测试

```bash
# 运行所有测试
pytest tests/ -v

# 运行单元测试
pytest tests/unit/ -v

# 运行测试并查看覆盖率
pytest tests/ -v --cov=src --cov-report=term-missing
```
